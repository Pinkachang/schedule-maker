# TPBL 賽程排程工具 — 使用與部署說明

一個純前端的賽程排程輔助工具。不需要後端、不需要安裝任何東西，一個 `index.html` 就能跑，可放上 GitHub Pages 永久免費使用，每季重複利用。

> 工具會把進度**自動存在你這台電腦的瀏覽器裡**（重整、關掉再開都不會消失）；要跨電腦或交給同事，再用「存檔」匯出 JSON 檔互傳。

---

## 一、放上 GitHub（每季只需設定一次）

1. 到 GitHub 建立一個新的 repository（例如命名 `tpbl-scheduler`），設為 Public。
2. 把 `index.html` 上傳到這個 repo（拖拉上傳即可）。
3. 進入 repo 的 **Settings → Pages**。
4. 在 **Source** 選擇 `Deploy from a branch`，Branch 選 `main`、資料夾選 `/ (root)`，按 Save。
5. 等一兩分鐘，頁面上方會出現網址，類似：
   `https://你的帳號.github.io/tpbl-scheduler/`
6. 把這個網址存成書籤，之後賽務組任何人開瀏覽器都能用。

> 之後要改工具本身（例如加功能），重新上傳 `index.html` 覆蓋即可，網址不變。

---

## 二、操作流程

工具分五個分頁：**排程／賽程預覽／進度總覽／隊伍與賽季／禁排與場館**。建議第一次照「隊伍與賽季 → 禁排與場館 → 排程」的順序設定。

### 隊伍與賽季
- 設定七隊的隊名、主場名稱、顏色、**隊徽 LOGO**。
- **LOGO**：每隊已內建官方隊徽，可直接點 LOGO 框上傳／更換（建議透明 PNG），或點右上角 × 移除改用顏色。新增球隊也能上傳，全程不用改程式。
- **主場名稱打一模一樣的兩隊會被自動視為「共用場館」**（例如國王、特攻都填「新莊體育館」），共用衝突檢查會自動啟用。
- 設定賽季起訖日，按「重新產生賽日」鋪出整季行事曆。
- 「可排時段」可自由增減。預設是週三/五 19:00、週六/日 14:30 與 17:00。
- **主場對戰輪數**可調（預設 3），總場次與各隊主客目標會自動跟著算。
- 頁底有「本機自動儲存」說明與「清除本機暫存」按鈕，以及「**清空所有賽程**」按鈕。
- **清空所有賽程**：一鍵清掉目前排好的所有比賽，並連帶刪除所有「協調中」的可用檔期（隊伍、賽季、時段、禁排、確定狀態的檔期都保留）。此動作不可復原，會跳確認框並提供「先下載備份 JSON」按鈕，建議清空前先備份。

### 禁排與場館
- **全聯盟禁排**：選日期區間＋原因（如世界盃），整段時間所有隊都不能排。
- **各隊禁排**：指定某隊某段時間不排賽。**可選三種類別**：
  - 主客皆禁（預設）：主場、客場都不排
  - 僅禁主場：不排主場，但仍可去別隊主場作客
  - 僅禁客場：不排客場，但自己主場照排
  - 清單上每筆都有下拉，可隨時改類別，不必刪掉重建。
- **場館可用檔期（球團回報）**：
  - 兩種方式並用：**①下載空白模板給球團填 Excel 後上傳**；**②直接在月曆上點選**該隊可用日期、批次加入（免整理 Excel）。
  - 系統採**白名單**邏輯：某隊還沒上傳檔期 → 主場到處能排；一旦上傳 → 該隊主場只能排在清單內的日期，其餘變「白名單外」。
  - **協調中標注**：每筆檔期可標「確定」或「協調中」（Excel 備註欄填「協調中」會自動辨識，清單上也能點切換）。排在協調中檔期的比賽，會在排程格、進度總覽、匯出 Excel 三處特別標示。

### 排程
- 行事曆以「週」為單位，每格是一個可排時段。
- 點空格 → 選主場、客場 → 確定。被鎖的時段不能點，被禁排的隊伍在選單會反灰。
- **白名單外主場**：如果某隊那天不在球團回報的可用檔期內，主場選單不會直接鎖死，而是顯示**橘色「白名單外」**可選。選了按確定會跳出小視窗，讓你把這場標成「確定」或「協調中」，系統會自動補一筆該隊的可用檔期再排入（適合「先卡位、之後再跟球團喬場地」）。
- **白名單色點提示**：空格下方會用小色點顯示「哪些隊這個時段可排主場」（隊伍顏色，協調中的半透明），滑鼠移上去看隊名，幫你決定空格排誰。
- 顏色判讀：虛線＝空檔、灰底＝鎖定、綠框＝已排、紅框＝有衝突、黃左框＝協調中、橘虛線＝白名單外。
- **即時場數計分板**（排程頁上方，捲動時固定在頂部）：每隊主客場數即時更新，免切分頁。左側可切換「精簡／詳細」，詳細模式顯示平日／假日各排了幾場主客；切換選擇會記住。
- **拖曳換時段**：同一天有多個時段時，按住已排的格子可拖到同日其他時段——目標有比賽就兩場對調、是空格就搬過去。只能在同一天內換（跨天會拒絕）。電腦滑鼠操作最順。
- **對戰表（右下角浮動鈕）**：排程時想看「目前對戰組合進度」「誰還沒打誰」「誰一週三戰太多」，點右下角「對戰表」叫出浮層，看完按 × 或 Esc 關閉。它浮在畫面上、不會動到你排程的位置，不用再切去進度總覽又捲回來。
- 點已排的格子可編輯或移除。

### 賽程預覽
- 把目前排出的賽程用乾淨的成品樣式呈現（比匯出 Excel 更好看），方便檢視或對外公告前確認。
- 全部球隊時依「週」分段；篩選單一球隊時自動切成該隊整季連續列表，並標出每場是主或客。
- 上方會提醒目前已排幾場、還沒排滿（草稿狀態）。

### 進度總覽
- 上方數字：已排場次、完成度、客隊待訂、協調中場次。
- **對戰組合矩陣**：每對組合主場目標 3 場，綠＝完成、黃＝未滿、灰＝0、紅＝超量。
- **各隊主客場數**：主、客各目標 18 場。
- **公平性檢視**：每隊平日（一～五）／假日（六、日）各排了幾場主客場、整季「一週三戰」幾次。同一欄落差大的會標色，方便檢視分配是否平均。
- **衝突與提醒清單**、**協調中清單**：每筆可按「前往」跳到行事曆對應位置。

## 三、衝突檢查規則

| 標示 | 規則 |
|------|------|
| 🔴 紅 | 同一場館同一週出現兩支不同主場隊（如新莊同週排了國王又排特攻） |
| 🔴 紅 | 週末主場未成對：某隊週六有主場，週日卻沒有同隊同時段主場（反之亦然） |
| 🔴 紅 | 客隊週末長途移動：某隊週六、週日連續作客，但兩天主場場館不同 |
| 🟡 黃 | 一週三戰：某隊單週出賽達 3 場 |

> 衝突只「標示」不「阻擋」——排程常需要彈性取捨，紅黃標示提醒你檢視，最終由你決定。

## 四、存檔與協作

- **自動存檔**：工具會自動把進度存在你這台電腦的瀏覽器裡，重整、關掉再開都還在，不用每次手動存。
  - 注意：本機暫存只綁**這台電腦、這個瀏覽器**。換電腦、換瀏覽器、清除瀏覽資料就會不見。
- **存檔（匯出 JSON）**：下載一個 `.json` 進度檔，用於跨電腦搬移、備份、交給同事。存檔時可在頂欄填「你的名字」，載入時會顯示這份是誰、何時存的。
- **載入（匯入 JSON）**：把 `.json` 讀回來繼續排。載入的 JSON 會覆蓋目前畫面。
- 多人協作就互傳這個 `.json`。

---

## 五、匯出

按右上「匯出 Excel」，得到 `總賽程` 格式（場次／日期／星期／時間／主場／客場／狀態），可直接接續後製。**狀態欄每場都會標明「確定」或「協調中」**（不留空白，方便用 COUNTIF、篩選、樞紐分析統計）。

---

## 已知限制與後續可擴充方向

- 目前是**輔助人工排程**，不做全自動求解（約束條件組合複雜，全自動不保證有解，軟性取捨也需人判斷）。
- 資料存在瀏覽器本機與 JSON，不上雲端（不需後端、不需帳號、避免資料庫安全問題）。多人看同一份目前靠互傳 JSON。
- 南北移動目前只檢查「客隊週末兩天場館一致」這條明確規則；更細的區域分群提醒可再擴充。
- 賽程預覽目前是畫面預覽，正式匯出仍走 Excel；若要列印／PDF 可再加。
