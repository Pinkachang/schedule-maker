# TPBL 賽程排程工具 — 使用與部署說明

一個純前端的賽程排程輔助工具。不需要後端、不需要安裝任何東西，一個 `index.html` 就能跑，可放上 GitHub Pages 永久免費使用，每季重複利用。

---

## 一、放上 GitHub（每季只需設定一次）

1. 到 GitHub 建立一個新的 repository（例如命名 `tpbl-scheduler`），設為 Public。
2. 把 `index.html` 上傳到這個 repo（拖拉上傳即可）。
3. 進入 repo 的 **Settings → Pages**。
4. 在 **Source** 選擇 `Deploy from a branch`，Branch 選 `main`、資料夾選 `/ (root)`，按 Save。
5. 等一兩分鐘，頁面上方會出現網址，類似：
   `https://你的帳號.github.io/tpbl-scheduler/`
6. 把這個網址存成書籤，之後賽務組任何人開瀏覽器都能用。

> 之後要改工具本身（例如加功能），重新上傳 `index.html` 覆蓋即可，網址不變。

---

## 二、操作流程

工具分四個分頁，建議照順序使用：

### 1. 隊伍與賽季
- 設定七隊的隊名、主場名稱、顏色。
- **主場名稱打一模一樣的兩隊會被自動視為「共用場館」**（例如國王、特攻都填「新莊體育館」），共用衝突檢查會自動啟用。
- 設定賽季起訖日，按「重新產生賽日」鋪出整季行事曆。
- 「可排時段」可自由增減。預設是週三/五 19:00、週六/日 14:30 與 17:00。臨時想開週一場、或改時間，在這裡加就好。

### 2. 禁排與場館
- **全聯盟禁排**：選日期區間＋原因（如世界盃），整段時間所有隊都不能排，行事曆會把那些格子鎖起來並顯示原因。
- **各隊禁排**：指定某隊某段時間不排主客場（如國家隊集訓）。該隊在那些日期的主、客場選項都會被鎖。
- **場館可用檔期（球團回報）**：
  - 先按「下載空白填寫模板」，得到一份每隊一頁的 Excel。
  - 寄給各球團，請他們填主場「**可以使用**」的日期與時段（整天可用就填「全天」）。
  - 收回後上傳，系統採**白名單**邏輯處理：
    - 某隊**還沒上傳**任何檔期 → 主場不受限制，到處都能排（避免新季資料還沒收齊就鎖死）。
    - 某隊**一旦上傳**檔期 → 該隊主場**只能**排在清單內的日期時段，其餘自動鎖定。
  - 客場不受此限制。可重複上傳多份，採累加。
  - **協調中標注**：每筆檔期可標為「確定」或「協調中」。球團在 Excel 備註欄填「協調中」會自動辨識；也可在清單上點狀態鈕切換，或手動新增時選擇。排在協調中檔期上的比賽，會在排程格子（黃色左框＋「協調中」標籤）、進度總覽（協調中場次統計＋專屬清單）、匯出 Excel（狀態欄標「協調中」）三處標示，方便追蹤哪些賽程還沒談定。

### 3. 排程
- 行事曆以「週」為單位，每格是一個可排時段。
- 點空格 → 選主場、客場 → 確定。被鎖的時段不能點，被禁排的隊伍在選單會反灰。
- 顏色判讀：虛線＝空檔、灰底＝鎖定、綠框＝已排、紅框＝有衝突。
- 點已排的格子可編輯或移除。

### 4. 進度總覽
- 上方四個數字：已排場次、完成度、剩餘、隊數。
- **對戰組合矩陣**：每對組合主場目標 3 場，綠＝完成、黃＝未滿、灰＝0。
- **各隊主客場數**：主、客各目標 18 場。
- **衝突與提醒清單**：每筆可按「前往」跳到行事曆對應位置。

## 三、衝突檢查規則

| 標示 | 規則 |
|------|------|
| 🔴 紅 | 同一場館同一週出現兩支不同主場隊（如新莊同週排了國王又排特攻） |
| 🔴 紅 | 週末主場未成對：某隊週六有主場，週日卻沒有同隊同時段主場（反之亦然） |
| 🔴 紅 | 客隊週末長途移動：某隊週六、週日連續作客，但兩天主場場館不同 |
| 🟡 黃 | 一週三戰：某隊單週出賽達 3 場 |

> 衝突只「標示」不「阻擋」——排程常需要彈性取捨，紅黃標示提醒你檢視，最終由你決定。

---

## 四、存檔與協作

- **存檔**：下載一個 `.json` 進度檔。
- **載入**：把 `.json` 讀回來繼續排。
- 多人協作就互傳這個 `.json`。工具本身不會自動記住資料（重整頁面會回到初始狀態），所以**離開前記得存檔**。

---

## 五、匯出

按右上「匯出 Excel」，得到 `總賽程` 格式（場次／日期／星期／時間／主場／客場），與你原本的工作表一致，可直接接續後製。

---

## 已知限制與後續可擴充方向

- 目前是**輔助人工排程**，不做全自動求解（你的約束條件組合複雜，全自動不保證有解，且軟性取捨本就需要人判斷）。
- 重整頁面不會自動保留資料，靠 JSON 存檔。若日後想要自動記憶，可再加（會用到瀏覽器本機儲存）。
- 南北移動目前只檢查「客隊週末兩天場館一致」這條明確規則；若想加入更細的區域分群提醒，可再擴充。
